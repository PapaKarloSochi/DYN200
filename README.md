## 📋 Описание

**DYN-200 Monitor** — профессиональное десктопное приложение для мониторинга, логирования и визуализации данных с датчиков крутящего момента DYN-200 (и аналогов ATO/CALT/DAYSENSOR DYN-серии) через интерфейс RS-485 с поддержкой протокола Modbus RTU.

Приложение разработано для промышленной эксплуатации и тестирования электродвигателей, генераторов и других вращающихся механизмов.

## ✨ Возможности

### 🔌 Подключение и коммуникация
- **Modbus RTU** — полноценная поддержка стандартного промышленного протокола
- **Автоопределение** режима работы датчика
- **Гибкая настройка** COM-порта, baudrate и slave-адреса
- **Валидация** входных данных и защита от некорректных настроек

### 📊 Визуализация данных
- **Три отдельных графика** для Torque, Speed и Power
- **Индивидуальное масштабирование** каждой оси (авто/фиксированный диапазон)
- **Крупные цифровые карточки** для отображения текущих значений
- **Тёмная тема** в стиле Material Design 3 для комфортной работы

### 🛡️ Безопасность и надёжность
- **Circuit Breaker** — защита от бесконечных попыток подключения
- **Retry Logic** с exponential backoff для восстановления соединения
- **Graceful Shutdown** — корректное завершение работы без потери данных
- **Thread-Safe** архитектура для безопасной работы с данными

### 📝 Логирование
- **CSV-экспорт** с настраиваемым путём сохранения
- **Ротация логов** с ограничением размера файла (5 MB)
- **Уровни логирования** (DEBUG, INFO, WARNING, ERROR, CRITICAL)
- **Цветовое кодирование** в интерфейсе

### ⚙️ Настройка и конфигурация
- **Коэффициенты пересчёта** для всех измерений
- **Настройка десятичных знаков** для отображения
- **Пределы осей графика** с возможностью автомасштабирования
- **Сохранение состояния** между сессиями

## 🖥️ Требования к системе

### Минимальные требования
- **Операционная система:** Windows 10/11, Linux, macOS
- **Python:** 3.10 или выше
- **RAM:** 4 GB
- **Диск:** 100 MB свободного места
- **Дисплей:** Разрешение 1280x800 или выше

### Аппаратные требования
- **RS-485 → USB конвертер** с гальванической развязкой
- **Датчик DYN-200** или совместимый
- **Блок питания** DC 24V (±10%), 0.2A

## 📖 Использование

### Быстрый старт

1. **Подключение датчика:**
   - Подключите датчик к COM-порту через RS-485 → USB конвертер
   - Убедитесь, что датчик включён и отображает данные на встроенном экране

2. **Настройка подключения:**
   - Откройте "Settings" → "Connection Settings..."
   - Выберите COM-порт из списка
   - Установите baudrate (по умолчанию 19200)
   - Нажмите "Подключить"

3. **Запуск измерений:**
   - Нажмите кнопку "ЗАПУСТИТЬ" для начала сбора данных
   - Данные отобразятся на графиках и в карточках значений

4. **Логирование данных:**
   - Нажмите "Логирование" и выберите файл для сохранения
   - Данные будут записываться в CSV-формате

## 🐛 Устранение неполадок

### Датчик не подключается

1. **Проверьте физическое подключение:**
   - Убедитесь, что кабель RS-485 правильно подключён (A→A, B→B)
   - Проверьте питание датчика (должен светиться дисплей)

2. **Проверьте COM-порт:**
   - Откройте Диспетчер устройств → Порты (COM и LPT)
   - Убедитесь, что конвертер RS-485 определён системой

3. **Проверьте настройки связи:**
   - Baudrate на датчике должен соответствовать настройкам в программе
   - Проверьте на дисплее датчика: обычно 19200 или 38400

### Нет данных на графике

1. Нажмите "ЗАПУСТИТЬ" после подключения
2. Проверьте, что датчик выдаёт данные (покрутите вал вручную)
3. Проверьте логи в файле `debug_log.txt`

### Ошибка "Permission denied" при логировании

- Убедитесь, что файл CSV не открыт в другой программе
- Проверьте права доступа к директории сохранения
- Используйте путь без специальных символов
